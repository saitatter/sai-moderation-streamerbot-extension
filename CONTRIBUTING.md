# Contributing

## Commit Messages
Use Conventional Commits:
- `feat:`
- `fix:`
- `refactor:`
- `ci:`
- `chore:`
- `docs:`
- `test:`

Examples:
- `feat(bridge): add moderation backend timeout policy`
- `fix(contract): align verdict casing with backend schema`

## Pull Requests
- PR title must follow Conventional Commits.
- Include validation notes (`dotnet test` output).
- Keep PR scope narrow and focused.

## Ownership
Changes require CODEOWNER review.
